``mmkv``\ 目前只支持一种Log方式：在执行命令前先写入\ ``MMBP`` request。
日志本身是异步写入的，因此不需要担心因等待IO准备而导致IO线程阻塞。

Config
======

在配置文件中可以指定存放位置和log方式。

Parser
======

由于request
log是二进制编码的，因此我预先准备了\ ``rlogdump``\ 来解析该文件并显示：

.. code:: shell

   ./build.sh rlogdump
   # ./release_build.sh rlogdump

``rlogdump``\ 除此之外也有一些选项对log文件做其他处理。
